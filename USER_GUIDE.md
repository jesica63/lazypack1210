## 這個工具可以做什麼？

### 🎯 兩大核心功能

**功能一：SEO 內鏈分析**
- 你寫好一篇文章後，系統會自動幫你找出適合加入內部連結的位置
- 就像有個 SEO 專家在旁邊告訴你：「這裡可以加個連結到我們的另一篇文章喔！」

**功能二：AI 懶人包生成**
- 你只需要提供主題和參考網址
- AI 會幫你讀完那些網頁，整理成一篇完整的懶人包文章
- 就像請了一位編輯幫你寫稿，還會標註資料來源

## 第一次使用：如何啟動系統

### ⚙️ 開啟系統需要「兩個步驟」

想像這個系統就像一家餐廳：
- **後台廚房**（API 伺服器）：負責處理資料
- **前台櫃檯**（網頁介面）：你跟系統互動的地方

兩個都要開，系統才能運作！

---

### 步驟 1️⃣：開啟後台廚房（API 伺服器）

**方法：**
1. 打開「終端機」程式（Terminal）
   - Mac 用戶：在 Launchpad 搜尋「終端機」

2. 輸入以下指令，然後按 Enter：
   ```bash
   cd /你的專案資料夾路徑/lazypack1210
   npm run server
   ```

3. 看到這個畫面就代表成功了：
   ```
   ========================================
   🚀 SEO LazyPack API 代理伺服器已啟動
   ========================================
   📡 監聽端口: 8787
   🔑 API Key: 已配置 ✅
   ```

**重要提醒：**
- ⚠️ 這個終端機視窗要保持開著，不能關閉！
- 💡 這就像餐廳的廚房，關了就沒辦法處理訂單了

---

### 步驟 2️⃣：開啟前台櫃檯（網頁介面）

**方法：**
1. 再開啟**另一個**終端機視窗
2. 輸入以下指令：
   ```bash
   cd /你的專案資料夾路徑/lazypack1210
   npm run dev
   ```

3. 看到這個訊息就成功了：
   ```
   ➜  Local:   http://localhost:5173/
   ```

4. 打開瀏覽器（Chrome 或 Safari），輸入上面顯示的網址
   - 通常是：`http://localhost:5173` 或 `http://localhost:3000`

**成功畫面：**
你會看到一個乾淨的介面，左上角寫著「SEO LazyPack 懶人包生成助理」

## 功能一：SEO 內鏈分析

### 🎯 使用時機
當你已經寫好一篇文章，想要加入內部連結來提升 SEO 時使用。

### 📝 操作步驟

#### 步驟 1：準備連結清單
在左上角的「Sitemap / 連結來源」區塊：
- 點擊「網址輸入」標籤
- 貼上你的內部連結清單（每行一個網址）

**範例：**
```
https://www.example.com/article1
https://www.example.com/article2
https://www.example.com/article3
```

💡 **小技巧：** 也可以貼上 Sitemap XML 網址，系統會自動抓取所有連結

#### 步驟 2：輸入文章內容
在左下方的編輯器：
- 直接打字，或
- 複製貼上你的文章，或
- 上傳 Word 檔案（點擊工具列的上傳按鈕）

#### 步驟 3：開始分析
點擊右上角的「**生成內鏈建議**」藍色按鈕

#### 步驟 4：查看結果
右側會顯示：
- **修訂後的文章**：已經幫你加好連結的完整文章
- **建議清單**：每個連結的說明
  - 錨點文字：用什麼文字加連結
  - 目標網址：連到哪裡
  - 選擇原因：為什麼選這個連結

#### 步驟 5：複製使用
- 點擊「複製 Markdown」或「複製 HTML」按鈕
- 貼到你的內容管理系統（WordPress、痞客邦等）

## 功能二：AI 懶人包生成

### 🎯 使用時機
當你需要針對某個主題寫一篇整理型文章，但不想從零開始時使用。

### 📝 操作步驟

#### 步驟 1：找到懶人包生成按鈕
在左下方的文章編輯器，工具列上有個**紫色漸層按鈕**：
```
🪄 懶人包生成
```

#### 步驟 2：填寫生成表單

點擊後會跳出「Deep Curation (懶人包生成器 v2.0)」視窗，需要填寫：

**必填欄位：**
- **主題**：你想寫的主題（例如：「人工智慧在醫療領域的應用」）
- **參考網址**：提供 2-6 個相關網頁連結（每行一個）

**選填欄位：**
- **開場白指示**：告訴 AI 你想要什麼樣的開場（例如：「用一個實際案例開場」）
- **大綱**：自訂段落標題（每行一個，留空系統會自動生成）

**填寫範例：**
```
主題：AI 醫療診斷的最新發展

開場白指示：用輕鬆的語氣介紹 AI 如何改變看病流程

大綱：
AI 診斷的原理是什麼
目前有哪些成功案例
醫生和病患的看法
未來發展趨勢

參考網址：
https://technews.tw/ai-medical/
https://www.commonhealth.com.tw/article/ai-diagnosis
https://health.udn.com/health/story/ai-medicine
```

#### 步驟 3：開始生成

點擊「**開始生成**」按鈕，然後耐心等待。

**處理流程：**
1. **資料蒐集助理正在閱讀網頁**（約 10-30 秒）
   - 系統正在讀取你提供的網址內容

2. **架構師正在整理資料填入大綱**（約 30-60 秒）
   - AI 正在分析資料，整理成段落大綱

3. **應用公司語態風格**（約 30-60 秒）
   - AI 將資料改寫成完整文章

💡 **小提醒：** 整個過程約需 1-3 分鐘，請不要關閉視窗

#### 步驟 4：查看與編輯

生成完成後：
- 文章會顯示在右側預覽區
- 也會同步到左側編輯器，方便你繼續修改
- 每個段落後面會標註資料來源

#### 步驟 5：匯出使用

點擊右側的「**複製 HTML**」或「**複製 Markdown**」按鈕

## 常見問題與解決方法

### ❓ 問題 1：打開瀏覽器顯示「無法連線」

**可能原因：** 後台伺服器沒有啟動

**解決方法：**
1. 檢查終端機是否有顯示「🚀 SEO LazyPack API 代理伺服器已啟動」
2. 如果沒有，回到「步驟 1️⃣：開啟後台廚房」重新執行

---

### ❓ 問題 2：點擊「生成內鏈建議」沒反應

**可能原因：** 內容或連結清單為空

**解決方法：**
1. 確認左下方編輯器有輸入文章內容
2. 確認左上方有輸入至少一個網址
3. 按鈕需要兩者都有才會變成藍色可點擊

---

### ❓ 問題 3：懶人包生成失敗，顯示錯誤訊息

**可能原因：** 網址無法讀取或網路問題

**解決方法：**
1. 確認提供的網址可以正常開啟（試著在瀏覽器打開看看）
2. 避免使用需要登入才能看的網頁
3. 至少提供 2 個以上的參考網址
4. 如果還是失敗，可以換其他網址試試看

---

### ❓ 問題 4：生成的內容不符合我的需求

**解決方法：**
1. **調整大綱**：在「大綱」欄位自訂段落標題，AI 會照著你的大綱寫
2. **更換參考網址**：提供更符合主題的網址
3. **修改開場白指示**：明確告訴 AI 你想要什麼風格（專業、輕鬆、新聞式等）

---

### ❓ 問題 5：想關閉系統

**正確關閉方式：**
1. 關閉瀏覽器分頁
2. 在兩個終端機視窗按 `Ctrl + C`
3. 關閉終端機視窗

**下次使用時：**
重新執行「步驟 1️⃣」和「步驟 2️⃣」即可

## 💡 使用小技巧

### ✨ SEO 內鏈分析技巧

1. **準備完整的連結清單**
   - 連結越多，AI 越能找到合適的內鏈
   - 建議至少 10 個以上

2. **文章長度要適中**
   - 太短的文章（少於 200 字）可能找不到合適位置加連結
   - 建議 500 字以上效果最好

3. **檢查建議的合理性**
   - AI 的建議不一定 100% 完美
   - 請用你的專業判斷，決定要不要採用

---

### ✨ 懶人包生成技巧

1. **主題要明確**
   - ❌ 不好：「AI」（太籠統）
   - ✅ 很好：「AI 在台灣醫療產業的應用現況」

2. **提供優質參考網址**
   - 選擇權威、內容豐富的網站
   - 避免農場文或廣告頁面

3. **善用自訂大綱**
   - 如果你已經有文章架構，填寫大綱能得到更符合需求的結果
   - 留空的話，AI 會自動生成 4 個段落

4. **生成後再編輯**
   - AI 生成的內容是「草稿」
   - 記得加入你的觀點和專業知識，讓文章更有深度
